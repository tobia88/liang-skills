Strategic Brief

Quest Body

Purpose

Remove all workflow assignment logic from the cartographer, making quest contracts workflow-agnostic. Contracts describe what to achieve and how to verify it, not which execution approach to use.

Desired Outcome

The cartographer produces quest contracts with no workflow field. Decomposition uses outcome boundaries with embedded verification. Quest ordering uses dependency topology with exposure tiebreaker.

Scope Boundary

Changes to liang-quest-cartographer/SKILL.md and its references/ directory only. No changes to tacticians, executors, or quest-core.

Victory Conditions

  • liang-quest-cartographer/SKILL.md contains no workflow assignment logic (no workflow detection, no workflow field in decomposition or validation)
  • Quest contract template (references/quest-contract-template.html) has no workflow field in the YAML comment block
  • Manifest template (references/manifest-template.html) does not include workflow pills for cartographer-generated output
  • Decomposition logic uses outcome boundaries with embedded verification instead of workflow-based splitting
  • Quest ordering uses dependency topology + exposure tiebreaker (risk × dependency fan-out)
  • Generated quest contracts are workflow-agnostic — any tactician or quick can plan them

Depends On

  • q001 — Establish Schema v2 in Quest Core

Boss Board / Risks

Fog of War / Open Questions
