Strategic Brief

Quest Body

Purpose

Ensure executors enforce workflow matching at the campaign level, hard-blocking and reporting mismatches, and catching planned-but-untagged quests as defense-in-depth layer 2.

Desired Outcome

Both executors (general and tdd) scan the campaign manifest, process only quests matching their workflow tag, hard-block and report on mismatches, and flag planned-but-untagged quests as a distinct error.

Scope Boundary

Changes to liang-quest-general-executor/SKILL.md and liang-quest-tdd-executor/SKILL.md only. No changes to how executors process quests internally beyond the filtering and blocking layer.

Victory Conditions

  • liang-quest-general-executor/SKILL.md processes only workflow: general quests from the manifest
  • liang-quest-tdd-executor/SKILL.md processes only workflow: tdd quests from the manifest
  • Both executors are campaign-level (assigned a campaign directory, not individual quests)
  • Both executors hard-block + report on workflow mismatch (not silent skip)
  • Both executors flag planned + no workflow as a distinct error category
  • The planned-but-untagged error is distinguishable from a workflow mismatch error

Depends On

  • q002 — Add Workflow Stamping to Downstream Skills

Boss Board / Risks

  • State machine complexity (Medium) — Executor logic must distinguish three states: matching workflow (process), mismatched workflow (hard-block + report), and missing workflow (distinct error). Getting the state machine wrong could cause silent skips.

Fog of War / Open Questions

  • Error message format for planned-but-untagged vs. mismatch cases
